# archie

Personal dotfiles, managed with [GNU Stow](https://www.gnu.org/software/stow/).
Each top-level directory mirrors the layout under `$HOME` and is stowed into
place by the `Makefile`.

## Install

```sh
make           # symlink everything into $HOME and link scripts into ~/.local/scripts
make clean     # unlink everything
```

Per-directory `*/.config/<name>/` content lands at `~/.config/<name>/`, and
each `*/.local/scripts/*` file is symlinked into `~/.local/scripts/`.

## Dependencies

System tools required by the Makefile and pre-commit hooks. Install on macOS
with `brew install ...`:

- `stow` — the symlink driver.
- `findutils` — provides `gfind` (BSD `find` lacks `-printf`). `make` errors
  early via the `preflight` target if this is missing.
- `sd` — used by `make snapshot` for in-place path rewriting.
- `lefthook` — git hook runner; wire it up with `lefthook install`.
- `shfmt` and `stylua` — format checks in the pre-commit hook.

Linux distros usually have GNU find as the default `find`, so `gfind` isn't
needed there.

## Pre-commit hooks

Defined in `lefthook.yml`. After cloning, run:

```sh
lefthook install
```

The hooks enforce:

- `shfmt -i=2 -d` on staged shell scripts under `zsh/`, `i3/.config/i3/scripts/`,
  `tmux/.config/tmux/scripts/`, `tmux/.local/scripts/`, and
  `snappy/.local/scripts/`.
- `stylua --check` on staged `*.lua`, using `stylua/.config/stylua/stylua.toml`.
- `make snapshot` plus `git diff --exit-code snapshot` — fails if the symlink
  layout drifts from the committed `snapshot` file.

## Snapshot

`snapshot` records the canonical mapping of `$HOME` paths to repo paths. It's
regenerated by `make snapshot`, which stows everything into `/tmp/archie-home`
and dumps the resulting symlink graph. The pre-commit hook enforces no drift,
so update it whenever you add or remove a stow directory.

## Host overrides

Machine-specific zsh config lives in `zsh/.config/zsh/overrides` (gitignored).
Source-time hook is in `zsh/.config/zsh/.zshrc`. Use this for secrets, host
aliases, and anything that shouldn't be committed. Keep the file at `chmod 600`
since it may contain credentials.
